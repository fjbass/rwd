Software Development with UML and Java

ECTS

10

Teacher

Allan Henriksen

Main purpose

The main purpose of the course is to provide students with the qualifications needed to understand the core object-oriented concepts and to implement smaller programs in Java from UML class diagrams.

Teaching methods and study activities

The semester has approx. 110 classroom lessons. A number of tests and assignments will be given during the course. One of these tests represent 20 % of the exam.

Resources

Tony Gaddis, "Starting Out with Java - early objects", 5th edition Addison-Wesley, 2015

  1. Identify the Java lexical structures: keywords, separators, operators, identifiers, literals and comments.
  2. Explain details of UML class diagrams.
  3. Identify selection and loop structures in UML activity diagrams.
  1. Construct Java programs with proper choice of selection and loop structures.
  2. Create and use objects in Java.
  3. Implement classes in Java using the object oriented concepts: encapsulation, inheritance and polymorphism.
  4. Implement one-to-one relations and differentiate between association, aggregation and composition.
  5. Implement one-to-many relations using array structures and a simple collection class.
  6. Implement exception handling for different types of exceptions.
  7. Implement persistence in text and binary files.
  8. Construct simple event-based GUI applications.
  9. Construct Java source code documentations.
  10. Interpret UML class diagrams, and construct corresponding Java code.
